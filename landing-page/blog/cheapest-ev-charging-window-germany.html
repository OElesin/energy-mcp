Find the Cheapest Hours to Charge Your EV in Germany

If you're on a dynamic electricity tariff in Germany — or thinking about switching to one — you already know the promise: charge your EV when spot prices are low, and save real money. The problem is finding those hours.

Electricity spot prices (EPEX Spot) change every hour. Tomorrow's prices are published at 14:00 CET. The cheapest hours shift daily depending on wind, solar, demand, and cross-border flows. Manually checking a price chart before bed every night is not a realistic long-term strategy.

The Old Way: 50 Lines of YAML Templates

If you use Home Assistant, the typical approach involves installing a price integration (Nordpool, ENTSO-E, or EPEX Spot via HACS), then writing complex Jinja templates to find the cheapest consecutive hours:

{% set prices = state_attr('sensor.nordpool', 'raw_today') + 
   state_attr('sensor.nordpool', 'raw_tomorrow') %}
{% set ns = namespace(best_start=none, best_avg=9999) %}
{% for i in range(prices | length - duration + 1) %}
  {% set window = prices[i:i+duration] %}
  {% set avg = window | map(attribute='value') | sum / duration %}
  {% if avg < ns.best_avg %}
    {% set ns.best_avg = avg %}
    {% set ns.best_start = window[0].start %}
  {% endif %}
{% endfor %}
{{ ns.best_start }}

This kind of template is fragile. It breaks at midnight when tomorrow's prices aren't available yet. It has timezone bugs during DST transitions. And it only tells you when to start — not how much it will cost.

The New Way: One API Call

Energy MCP provides a find_cheapest_window tool that does the same thing in a single request:

POST https://energy-mcp.getbrechtai.com/mcp

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "find_cheapest_window",
    "arguments": {
      "duration_hours": 3,
      "consumption_kwh": 11
    }
  }
}

Response:

{
  "duration_hours": 3,
  "start": "2026-07-20T02:00:00+02:00",
  "end": "2026-07-20T05:00:00+02:00",
  "avg_price_ct_kwh": 4.07,
  "total_cost_eur": 0.45,
  "consumption_kwh": 11.0,
  "prices": [...]
}
What you get: The optimal start and end time, the average price during that window, and the estimated total cost for your charging session. Updated hourly with fresh EPEX Spot data.

How Much Can You Actually Save?

On a typical German day, spot prices vary between 3–16 ct/kWh. If your EV needs 11 kWh (about 60 km of range), the difference between charging at the cheapest vs. most expensive hour can be €0.45 vs. €1.83 — that's 4x.

Over a year of daily commute charging, optimizing your window can save €300–500 compared to always charging at 6 PM when you get home (typically the most expensive hour).

Use It in Home Assistant

No custom integration needed. A command line sensor calls the API every 30 minutes:

command_line:
  - sensor:
      name: cheapest_ev_window
      command: >
        curl -s -X POST https://energy-mcp.getbrechtai.com/mcp
        -H "Content-Type: application/json"
        -H "Accept: application/json, text/event-stream"
        -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"find_cheapest_window","arguments":{"duration_hours":3,"consumption_kwh":11}}}'
        | grep "^data:" | sed 's/^data: //'
      value_template: >
        {{ (value_json.result.content[0].text | from_json).start }}
      scan_interval: 1800

Then trigger your wallbox with an automation:

automation:
  - alias: "Start EV charging at cheapest window"
    trigger:
      - platform: template
        value_template: >
          {{ now().isoformat()[:16] ==
             states('sensor.cheapest_ev_window')[:16] }}
    action:
      - service: switch.turn_on
        target:
          entity_id: switch.ev_charger

How It Works Under the Hood

Energy MCP fetches hourly day-ahead prices from aWATTar, which provides free access to EPEX Spot market data for Germany. Prices for the next day are published daily at 14:00 CET.

The find_cheapest_window algorithm slides a window of N consecutive hours across all available prices (today + tomorrow = up to 48 hours), calculates the average price per window, and returns the window with the lowest average. Total cost is computed from your specified kWh consumption.

  • Data source: EPEX Spot via aWATTar (free, no API key)
  • Update frequency: Hourly, with next-day prices from 14:00 CET
  • Latency: ~10ms (Rust on AWS Lambda)
  • Coverage: Germany and Austria
